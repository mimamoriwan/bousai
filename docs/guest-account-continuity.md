# ゲスト記録とGoogle連携の方針

## 目的

みまもりWANは登録なしですぐに使えることを基本とし、投稿・お散歩記録・マイマップなどをゲストでも利用できるようにします。Google連携は、詳しいプロフィール、MYルート、機種変更時の引き継ぎなどを追加する手段として扱います。

## 新しいGoogleアカウントを連携する場合

Firebaseの匿名アカウントへGoogle認証を連携します。連携の前後で利用者ID（UID）は変わらないため、UIDに紐づく次の記録をそのまま利用できます。

- 投稿と非公開投稿
- マイマップの保存状態
- お散歩アクション
- プロフィール
- 画像の保存領域

この動作はFirebase Emulatorの自動テストで継続的に確認します。

## すでに登録済みのGoogleアカウントを選んだ場合

ゲストから既存のGoogleアカウントへ自動的に切り替えると、匿名UIDに紐づく記録が画面から見えなくなるため、自動ログインは行いません。

- 現在のゲストセッションと記録を維持する
- マイページに「ゲスト記録を保護しました」と表示する
- 既存アカウントとの統合は、記録の所有権を確認できる安全な移行機能を用意してから対応する

## 今後の課題

異なるUID間の記録統合には、クライアントだけでなく信頼できるサーバー側での本人確認と一括移行が必要です。統合処理の設計とテストが完成するまで、既存アカウントへの自動移行は実装しません。
