# 🐶 みまもりWAN（旧：つくばペット防災）

みんなの歩幅で作る、地域の安全とペットに優しいまちづくり。
お散歩中の危険箇所や新しい発見から、いざという時の防災・避難所情報までを、スマートに共有できる地域密着型のPWAアプリです。

## 💡 アプリ名とコンセプト

**「みまもりWAN」**
「愛犬がお散歩中に何でも口にしてしまう危険」を防ぎたい、という想いから生まれました。
地域の飼い主さんたちを「みまもり隊員」とし、『知っていれば予防できる』リアルな安全情報（ガラスの破片が落ちている、除草剤が撒かれていた等）や防災情報を、日々のお散歩を通じて共有するサービスです。

## 🛠 使用技術（Tech Stack）

### フロントエンド
- **React**: UIライブラリ（コンポーネントベースの開発）
- **Vite**: 高速なビルドツールとローカルサーバー
- **React Router**: SPA（シングルページアプリケーション）のシームレスな画面遷移
- **Leaflet / React-Leaflet**: オープンソースのインタラクティブマップの実装
- **Vanilla CSS**: 軽量かつ柔軟なスタイリングシステム

### バックエンド（BaaS）
- **Firebase Authentication**: Googleアカウントを利用したユーザー本登録およびゲスト（匿名）ログイン
- **Cloud Firestore**: リアルタイムなデータベース同期とデータの永続化
- **Cloud Storage**: HEIC変換・圧縮後のスポット画像および愛犬プロフィール写真のアップロード・保管
- **Firebase Hosting**: プロダクトの高速な配信・公開とデプロイ環境

## ✨ 実装済みの主要機能と仕様

### コア機能・マップ関連
- **マップピンの48時間自動アーカイブ**: 投稿されたピン（「危険・注意」「お散歩情報」）は、情報の鮮度を保つため48時間経過後に自動で非表示になります。（※「防災・避難所」カテゴリは例外として常時保持・表示されます）
- **避難所フィルター**: 公式の避難所ピンは、初期状態では非表示になっており、フィルターで「🎒防災・避難所」を選択した時のみ表示されます。
- **「💖ありがとう」トグル機能**: マップのピン詳細ポップアップおよびリストビューの両方から直接「ありがとう」ボタンを押せます。ゲスト（匿名）ユーザーも利用可能で、1ユーザーにつき1回までのトグル動作となります。
- **マップ連動・最新情報フィード**: 投稿されたスポットをマップ下部にタイムライン表示し、タップでマップ上の対象ピンへ自動移動します。

### ユーザー・コミュニティ機能
- **ゲストログイン・本登録**: 登録不要ですぐに機能を使えるゲスト（匿名）ログイン機能と、Googleアカウントによる本登録機能を提供。
- **隊員ナンバーの自動発行**: ログイン（本登録）時にFirestoreのトランザクションを用いてシーケンシャルな「隊員ナンバー（連番）」を自動発行し、アプリのヘッダーに表示（例：No.001）します。
- **投稿者情報の匿名化**: ピンの詳細ポップアップには、個人情報（名前やアイコン）の代わりに「投稿者：みまもり隊員 No.XXX」または「ゲスト隊員」として表示し、ユーザーのプライバシーを保護します。
- **愛犬プロフィールと写真アップロード**: マイ・ペット手帳（マイページ）にて、愛犬の写真アップロードが可能。FirestoreとStorageを用いて保存・リサイズ表示されます。

### UI/UX改善・ネイティブアプリライクな体験
- **マップおよびリストの独立スクロール**: マップ本体のサイズを固定し、画面全体がバウンス（意図しないスクロール）するiOS特有の挙動を完全に防止。リスト領域のみが滑らかに縦スクロールするデザインを採用しました。
- **ボトムナビゲーションの干渉防止**: リスト最下部までスクロールした際に、ナビゲーションメニューが投稿コンテンツに被らないよう余白（パディング）を維持しています。
- **Pull-to-Refresh & 手動更新**: リスト領域での「引っ張って更新」と、見出し横の「🔄」リロードボタンを用いた最新の周辺データ（ピン・リスト）の再取得に対応しました。
- **スプラッシュスクリーン**: PWA起動時（React初期ロード前）の真っ白な画面を防ぐため、テーマカラーに合わせたCSSスピナー（ローディングUI）を表示します。
- **画像プレビューの自動調整**: マップポップアップ内の添付写真が大きすぎる場合でも適切なサイズに自動縮小され、UIの崩れを防ぎます。

### ソーシャルシェア（OGP）対応
- **リッチプレビューの表示**: LINE等でURLがシェアされた際に、本アプリ専用のサムネイルアイコン画像（肉球マーク）とタイトルがカード形式で表示されるよう、OGP設定およびTwitter Cardのマークアップを `index.html` に実装しています。

## ⚠️ 開発者向け特記事項（重要）

本システムの運用における、データベース（Firestore / Storage）のセキュリティルールおよび移行作業に関する注意事項です。

1. **セキュリティルールの必須設定**
   - ゲスト（匿名）ユーザーであってもマップ投稿や画像のアップロード、「ありがとう」の更新ができるよう、認証済み（`request.auth != null`）を許可するルール設定が必要です。
   - `users` コレクションのマイページ編集（本人確認）や、`metadata/counters` を用いた隊員ナンバー発番処理のパーミッション設定に注意してください。
2. **既存ユーザーの隊員ナンバー手動付与（マイグレーション）**
   - 自動発番機能のリリースより前に登録されていた既存のGoogleユーザーに対しては、手動またはスクリプトを用いてFirestore上で任意の `memberNumber` を付与し、かつ `metadata/counters` の数値を同期するマイグレーション作業が必要です。

## 📂 主要なディレクトリ・ファイル構成

プロジェクトのフロントエンド機能は `src/` 配下に集約されており、以下の役割を持っています。

```text
src/
├── App.jsx                 # アプリのメインのルーティング定義（React Router）
├── firebase.js             # Firebase製品（Auth, Firestore, Storage）の初期化とエクスポート
├── index.css               # グローバルなデザイン定義、CSS変数、ベースコンポーネントのスタイル
├── context/
│   └── AuthContext.jsx     # ユーザー認証・隊員ナンバー採番処理などのContext API
├── components/
│   ├── Layout.jsx          # ヘッダー（隊員No等の表示）とボトムナビゲーションを含む構造
│   └── Layout.css          # Layout専用のスタイリング
├── pages/
│   ├── Map.jsx             # [🗺️マップ] 地図描画、ピンのフィルター、自動アーカイブ、リスト連動アクション
│   ├── Guide.jsx           # [📖ガイド] アプリの使い方と防災情報の折りたたみ表示
│   └── Profile.jsx         # [🐾マイページ] Google認証、プロフィール写真管理、フィードバック機能
├── utils/
│   └── imageUtils.js       # ファイルのHEIC形式対応や、Canvasによる画像圧縮処理
└── data/
    ├── shelters.json       # 避難所の初期マスターデータ
    └── guides.json         # アコーディオン内に表示する防災テキストデータ
```
